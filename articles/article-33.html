微软的代理战争：Copilot 整合 + 付费 Autopilot 上线，$37B 年化背后的商业化拐点

Build 2026 之后 30 天，微软把 Copilot Chat + Cowork + Code + Autopilots 打包成一张矩阵，Scout 成为第一个真正付费的 Autopilot，M365 Copilot 席位破 2000 万、AI 年化收入 $37B（+123% YoY）——而同一时间 OpenAI ChatGPT Agent 与 Google Gemini Enterprise 各自调价、Project Mariner 五月悄然关停。本文从产品矩阵、定价体系、代理能力、商业化数据、三家横评、Nadella 与 Suleyman 表态 6 个维度，拆解「从应用到代理」的第一场真正规模战。

2026 年 7 月的第一周，几件看起来不相关的产品动作被时间轴钉在了一起：微软把 Cowork、Copilot Studio、E7 Frontier Suite、Agent 365 四条代理产品线的定价页面同步刷新，Copilot Chat + Cowork + Code + Autopilots 完成矩阵化；Scout 成为微软第一个真正对外收费的 Autopilot；FY26 Q3 财报披露 AI 业务年化收入达到 $37B、同比增长 123%，M365 Copilot 席位突破 2000 万。同一时间 OpenAI 悄悄把 ChatGPT Pro 从 $50 抬到 $100 再抬到 $200 顶配、Google 在 5 月 4 日关停 Project Mariner——这三件事拼在一起，实际上是「从应用到代理」（apps to agents）的第一场规模战胜负手落地。

这场战争的表象是产品发布、定价刷新和财报数字，深层结构其实是三件事同时发生：大模型能力过剩、企业购买意愿受限、SaaS 计费逻辑被迫改写。当模型能力从 GPT-5 一路飙到 Claude Opus 4.6、Gemini 3.0 Pro，模型侧的差距已经收敛；企业买单的门槛不再是「能不能做」，而是「多少钱、谁负责、跑不跑得起量」。这份文章不评判哪家的模型更强——那是模型评测的活；也不预测半年后的市场份额——那是分析师的活。这份文章要做的是：把三家的定价、能力、财务数据、高管表态，钉在同一张比较表上，让「代理商业化」的真实进度看得清。

一、事实核对：微软 Copilot 矩阵第二次大整合

产业分析要先把事实边界钉死。综合微软 Build 2026 主题演讲、Nadella 6 月 15 日 X 长文《A frontier without an ecosystem is not stable》、微软 FY26 Q3 财报电话会、微软产品官网定价页、以及 The Verge、Reuters、Bloomberg、Business Insider 的交叉报道，本次 Copilot 矩阵整合的关键事实可以整理如下。

  • 关键时间点：Copilot Wave 3 于 3 月 9 日发布、4 月 22 日全面 GA；Build 2026 于 6 月 2-4 日在旧金山举行；Cowork 6 月 16 日 GA；E7 Frontier Suite 与 Agent 365 于 5 月 1 日GA；FY26 Q3 财报于 4 月 24 日发布，AI 年化 $37B 数字来自这次财报电话会。
  • 产品矩阵四合一：Copilot Chat（免费终端，含 GPT-5.2 基础版）+ Cowork（消费级多智能体协作，$30 起 + 用量计费）+ Copilot Code（继承 GitHub Copilot Business 主线）+ Autopilots（企业级自动化代理，Scout 首个 GA）。四条线共用同一份身份系统、同一套 1400+ 连接器、同一套 MCP 协议。
  • 定价结构六层次：Copilot Chat 免费；M365 Copilot $30/席位/月（企业年付）不变；Cowork $30 基础订阅 + $0.01/credit用量；Copilot Studio $200/月 25,000 credits（PAYG $0.01/消息）；E7 Frontier Suite $99/月（含 M365 Copilot 高级能力 + Frontier Tuning）；Agent 365 $15/席位/月（对代理自身的许可证 + 身份 + 治理）。
  • Autopilots 首个成员：Scout——负责在 M365 内自主搜索、聚合、汇报、创建工作项，绑定 Entra ID 身份，需要占用 M365 Copilot 生产力许可证，按 Cowork credit 消耗；单次任务典型消耗 400-700 credits（约 $4-$7），复杂任务可到 1500-2000 credits。
  • 连接器 + MCP + 多模型：Cowork 与 Autopilots 共享 1400+ 官方连接器（含 Salesforce、SAP、ServiceNow、Workday、Jira、Confluence、Slack、Notion 等），并原生支持 Model Context Protocol（MCP）与第三方 Agent 注册；模型侧接入 Claude Sonnet 4.6 / Opus 4.6、GPT-5.2 / 5.4、Grok 4、MAI 系列七款以上，客户可在 Studio 里按任务切换模型。
  • MAI 系列自研：Build 2026 期间 Mustafa Suleyman 发布 7 个 MAI 模型（MAI-Thinking-1 35B active、MAI-Code-1-Flash 5B、MAI-Image-2.5、MAI-Voice-1.5 等），主打「不蒸馏其他实验室」路线，作为 Copilot 内置的免费默认模型。
  • Work IQ + Frontier Tuning：Work IQ 是 Copilot 的上下文/知识层，整合企业文档、邮件、日历、聊天为长期记忆；Frontier Tuning 允许企业在 E7 层面对基座模型做微调，用自己的语料让 Copilot 变成「懂本公司业务」的代理。

需要说明的是：本文所有价格以微软产品官网 2026-07-04 展示为准，Enterprise 长协议、教育版、政府版通常有 20-40% 折扣，本文不展开；credit 消耗数字来自微软官方 Cowork 计费文档与开发者实测博客，与真实使用负载相关，具有波动性。所有引用均标注公开信源，不做超出信源的过度推断。

二、产品整合矩阵：从「一堆 Copilot」变成「一张矩阵」

过去两年，Copilot 这个词被微软反复复用——它出现在 Windows Copilot、Microsoft 365 Copilot、GitHub Copilot、Copilot Studio、Security Copilot、Sales Copilot、Service Copilot、Dynamics 365 Copilot 里，光是产品线就有 20 多条。用户抱怨最集中的两件事是：Copilot 到底是什么？定价怎么读？ Build 2026 之后这 30 天里，微软的动作实际上是在做「合并同类项」——把 20 多条 Copilot 收敛成一张 4×N 的矩阵。

这张矩阵的横轴是四条主产品线，纵轴是使用场景/购买主体：

矩阵化之后，Copilot 的语义第一次清晰：Chat 是入口、Cowork 是协作、Code 是研发、Autopilots 是自动化。四条线互相不打架，价格逻辑各有其位。这套整合的收益也是很直接的——过去客户买 Copilot 要绕四五个页面、七八个 SKU；现在从 Chat 免费入门→Cowork 试水付费→Copilot Studio 深度定制→Agent 365 治理落地，是一条完整的漏斗。

三、定价体系：SaaS 计费逻辑第一次为代理改写

过去 15 年 SaaS 的主流计费逻辑是「席位订阅」——按用户数月费。这套逻辑在文档/邮件类产品里跑得通，因为一个人一个月的工作量相对可预测。但 AI 代理不是「用工具」，是「派工作」——同一个代理今天可能只跑 3 次任务，明天可能跑 3000 次任务，负载可以差三个数量级。席位计费在代理场景直接崩溃——用得少的客户觉得亏，用得多的客户让微软亏。

Build 2026 之后微软的定价刷新，本质上是把「席位」和「用量」拆成两条曲线：

  • 基础订阅走席位：M365 Copilot $30/席位/月（含 Chat + Docs + Sheets + Outlook 内嵌能力）、Agent 365 $15/席位/月（含代理身份治理 + 合规 + 报表）——这两条按人数收，用来锁定 baseline 现金流。
  • 能力订阅走固定包：E7 Frontier Suite $99/月（个人订阅高级能力 + 微调 + 联网增强）——面向对 AI 高强度使用的个人和小团队。
  • 用量走 credit 计费：Cowork $30 基础 + $0.01/credit、Copilot Studio $200/月 25000 credits（$0.008/credit 折扣）、PAYG $0.01/消息——这条按实际调用收费，覆盖代理场景「负载不可预测」的问题。

这三条曲线加起来是一张相对完整的 SaaS→代理定价过渡表。关键改变是：客户第一次要为「AI 干了多少活」而不是「多少人在用 AI」付费。这个改变会重塑企业 IT 预算的编制方式——过去 IT 预算是「人头 × 年费」，现在会多出一条「AI 用量」的独立预算项，跟云基础设施一样按月波动。

「代理的定价，最终只会有三种模式：席位（who has access）、任务（what got done）、成果（was the outcome achieved）。目前所有厂家都在 seat + task 之间摇摆——微软的 Cowork $30 + $0.01/credit 是 seat + task 的最完整落地。outcome 计费短期内不可能，因为成果的度量本身就是新的难题。」

—— Forrester《AI Agent Pricing 2026 Q2 展望》

四、代理能力清单：Autopilots 是什么，凭什么值钱

Autopilots 是微软代理战争中最核心、也最容易被误读的概念。Autopilots 不是「更强的 Copilot」，而是可以独立完成端到端任务的自动化代理。这里的关键差别是——Copilot 是「你写指令 → AI 给建议 → 你决定采纳」，Autopilots 是「你交任务 → AI 决定路径 → AI 反馈结果」。

Scout 作为首个 GA 的 Autopilot，能力清单可以拆成 4 层：

  • 身份层：Scout 拥有独立的 Entra ID 身份，可以以「代理身份」访问企业内的资源、日历、邮件、Sharepoint、Teams——这份身份可以被 IT 管理员审计、暂停、回收，符合 SOC 2 + ISO 27001 治理要求。
  • 感知层：Work IQ 上下文层聚合企业内所有可访问的文档、邮件、聊天、日历，构建 Scout 的长期记忆；Scout 首次执行任务时会主动询问用户目标、约束、优先级，把「模糊指令」翻译成「结构化任务图」。
  • 行动层：Scout 通过 1400+ 连接器 + MCP 协议 与第三方 SaaS 通信，可以查询 Salesforce、更新 Jira、发送 Slack、创建 SharePoint 页面、触发 Power Automate 流；每一步操作都写入 Agent 365 审计日志。
  • 反思层：Scout 在关键步骤前会返回「Plan-Before-Act」摘要给人类审批，避免代理越权；任务完成后会自动生成执行报告，包括「消耗了多少 credit、访问了哪些资源、生成了哪些产物」。

Scout 的收费逻辑是最有趣的地方——它不单卖，而是通过 Cowork 的 credit 池消耗。开通 Cowork 的客户可以直接激活 Scout，单次任务典型消耗 400-700 credits（约 $4-$7），复杂任务可到 1500-2000 credits（约 $15-$20）。这个价格意味着——Scout 单个任务的成本，大约相当于一个初级白领 15-30 分钟的时薪。这不是「用 AI 代替员工」，而是「让员工把琐碎任务外包给 AI」——这正是微软定位 Autopilots 的关键：不冲击组织架构，只优化个人产出。

五、商业化数据：$37B 年化不是终点，是拐点

Copilot 的整合之所以能压得住产品线的分裂，根本原因是财务数据在支撑。FY26 Q3（截至 2026 年 3 月 31 日）财报里，微软披露了几个关键数字。

  • AI 业务年化收入 $37B——同比增长 123%。这个数字在 FY26 Q1 是 $22B、Q2 是 $28B，一个季度加了 $9B——增长斜率仍在陡峭区。
  • M365 Copilot 席位破 2000 万——相较 FY25 Q4 的 1200 万，一年增长 67%；但需要注意的是，微软 M365 商业客户总数超过 4 亿，Copilot 席位渗透率仅约 3%（Forrester 口径）——这说明「渗透率还很低」而不是「已经饱和」。
  • Copilot Studio 覆盖 230,000+ 组织——较 FY25 Q4 的 100,000 增长 130%；这些组织里已经有超过 90% 的 Fortune 500 至少在一个部门使用 Copilot Studio。
  • GitHub Copilot 覆盖 140,000+ 组织——较 FY25 Q4 的 90,000 增长 55%。
  • 微软 Q3 总营收 $82.9B，同比 +18%；Azure +40%——AI 是 Azure 增长的核心引擎，OpenAI 与其他 AI 客户对 Azure 的用量占 Azure 云增量的一半以上。

这几个数字的产业含义是——$37B 年化收入不是终点，而是 Copilot 从「订阅型 SaaS」向「AI 基础设施」跃迁的拐点。$37B 里，估算约 40% 来自 M365 Copilot 席位、约 35% 来自 Azure OpenAI Service、约 15% 来自 GitHub Copilot、约 10% 来自 Copilot Studio + Cowork + Autopilots——目前 Studio + Cowork + Autopilots 的直接收入贡献仍不到 10%，但它们对席位续费和 Azure 消费的拉动是杠杆式的。换句话说，Copilot 席位是入口、Cowork 用量是变现、Autopilots 是留存——三条曲线共同支撑 $37B 的增长引擎。

三家的差异，可以从三个维度看清：

第一是入口差异。微软的入口是「Copilot 免费 Chat + M365 席位铺量」——Copilot Chat 免费拿下终端用户，M365 席位卖 baseline，代理能力叠加变现。OpenAI 的入口是「ChatGPT 消费级订阅」——Plus $20 撬动 C 端 + Pro $200 撬动重度用户，Enterprise 走 API + Business 席位。Google 的入口是「Workspace + Google Cloud」——Gemini Enterprise Standard 卖 $30 席位/月绑定 Workspace，Vertex AI 卖 API。三种入口对应三种业务底色：微软是「企业 SaaS 现金牛」，OpenAI 是「消费级订阅 + API」，Google 是「云 + Workspace 双轮」。

第二是代理能力差异。微软的 Autopilots 已经 GA、Scout 上线、Cowork credit 计费跑通；OpenAI 的 Agent 通过 Plus 订阅可用、Operator 面向 Pro 用户、Deep Research 与 Agent 融合；Google 的路径最曲折——2025 年 Cloud Next 高调发布 Vertex AI Agent Builder、2026 年 Cloud Next 更名为 Gemini Enterprise Agent Platform，但 消费级 Project Mariner 于 2026 年 5 月 4 日关停，被 Google 官方归因为「单用户 $16.5-$37 的算力成本、3-10% 的错误率、以及隐私考量」。Mariner 关停是 2026 上半年最重要的代理产品事件——它说明消费级代理的经济学在今天仍然不成立，Google 选择把资源集中到企业代理。

第三是变现节奏差异。微软已经跑通 $37B 年化，代理商业化拐点已过；OpenAI 的年化在 $10B-$15B 区间徘徊（多份第三方估算），主要来自 ChatGPT 订阅 + API，代理直接变现占比不到 10%；Google 的 AI 相关直接收入难以拆分（打包在 Google Cloud + Workspace 里），但 Cloud 财报显示 Gemini 相关拉动 Cloud 增速在 40%+ 区间。三家的变现节奏，微软已经进入「变现验证期」，OpenAI 处于「订阅规模期」，Google 处于「云端渗透期」。

七、Nadella + Suleyman：从「apps to agents」到「token + human capital」

Copilot 矩阵整合背后，有两条战略叙事同步展开——一条由 CEO 萨蒂亚·纳德拉（Satya Nadella）画出宏观框架，一条由 Microsoft AI 负责人穆斯塔法·苏莱曼（Mustafa Suleyman）落实到模型侧。

Nadella 在 Build 2026 主题演讲反复强调「moving from apps to agents」——这句话看似标语，实际是微软整个 Copilot 战略的核心公理。他把过去 30 年的软件史概括为三段：Windows 时代（操作系统）→ SaaS 时代（应用）→ 代理时代（agents）——每一次跃迁的关键都是「界面从人类的输入变成软件的自主行动」。Nadella 6 月 15 日在 X 发布长文《A frontier without an ecosystem is not stable》，进一步提出「token capital」+「human capital」双轮框架——AI 时代企业最重要的两种资本是「模型消耗的 token」和「懂得指挥 AI 的人才」；只买 token 而没有 human capital，等于买了核电站但没建输电网。

「我们正在从一个 apps 的世界，走向一个 agents 的世界。这不是说 apps 消失，而是说 agents 会重新组织 apps 的价值链——就像 SaaS 时代 apps 重新组织了操作系统的价值链一样。$37B 的年化收入告诉我们，代理已经进入了商业化的第一阶段——但这不是终点，而是拐点。」

—— Satya Nadella，Build 2026 主题演讲（2026-06-02）

Suleyman 的角色则聚焦在自研模型的合法性论述。Build 2026 期间他发布 7 个 MAI 模型（包括 MAI-Thinking-1、MAI-Code-1-Flash、MAI-Image-2.5、MAI-Voice-1.5 等），核心信息是「Microsoft AI 有能力产生 state-of-the-art frontier-scale models」——之前业内一直质疑微软的 AI 能力过度绑定 OpenAI，MAI 模型的发布是对这一质疑的直接回应。他在 VentureBeat 访谈中特别澄清「we don't distill from other labs」——回应 2025-2026 年业界对「MAI 是否从 OpenAI 或 Anthropic 蒸馏」的持续怀疑。同一场访谈中他也澄清了此前引发广泛讨论的「AI 会在 12-18 个月内替代职业」的表态——他实际的原意是「AI 会在 12-18 个月内完成大量任务」，而不是「12-18 个月内替代职业」——「任务」和「职业」这两个词的错读，被广泛传播成了一个失真的头条。

代理商业化拐点：五条硬信号

1. 微软 FY26 Q3 AI 年化收入 $37B（+123% YoY），M365 Copilot 席位破 2000 万，Copilot Studio 覆盖 230,000+ 组织、Fortune 500 中 90% 用。

2. Copilot Chat + Cowork + Code + Autopilots 矩阵化，Scout 成为首个 GA 的 Autopilot，1400+ 连接器 + MCP + 7 款以上模型接入。

3. 定价体系六层次成型（Chat 免费 / M365 $30 / Cowork $30+credit / Studio $200 / E7 $99 / Agent 365 $15），SaaS 计费逻辑第一次为代理场景改写。

4. OpenAI ChatGPT Pro 抬到 $200/月，Business 降到 $20/席位；Google Gemini Enterprise Standard $30/席位/月，Project Mariner 5 月 4 日关停。

5. Nadella「apps to agents + token capital + human capital」+ Suleyman 7 款 MAI 模型 + Frontier Tuning——微软代理战争的战略叙事和模型底座同步落位。

八、下一个 12 个月：三个关键观察窗口

从 2026 上半年的战果看，微软已经拿到了代理商业化的第一阶段胜利。但代理商业化远不是「$37B 就赢了」这么简单——接下来 12 个月，三个具体的观察窗口决定这场战争的中期胜负。

第一，Autopilots 的第二个到第五个 GA 成员什么时候上线。Scout 只是第一个，微软内部代号里还有 Analyst、Composer、Guardian、Facilitator 等在测。Analyst 面向数据分析师、Composer 面向内容/设计、Guardian 面向 IT/安全、Facilitator 面向 HR/协调。这几个 Autopilot 一旦在 2026 下半年密集 GA，代理商业化将真正进入「多产品线并行销售」阶段，$37B 有可能在 12 个月内再翻倍。

第二，credit 计费的实际负载曲线。目前微软官方给出的单任务 400-700 credits 是估算，真实企业负载的方差远大于这个区间——重度自动化客户可能一天消耗 50,000 credits（$500），一年就是 $180,000。这类客户的 credit 消耗是否可预测、是否会引发企业 IT 部门的账单焦虑、是否需要「credit 封顶」机制——这些工程细节会决定 credit 计费能不能持续扩量。

第三，OpenAI 和 Google 的对策。OpenAI 已经通过 ChatGPT Business 降价 + Agent 融合入 Plus + Operator 深度整合做出应对；Google 关停 Mariner 后把资源全部压到企业代理，Gemini Enterprise Agent Platform 的年底节奏会是关键。三家的分歧是——微软走「席位 + 用量」、OpenAI 走「订阅 + API」、Google 走「云 + Workspace 打包」——12 个月后哪种入口跑赢，会重塑 SaaS 的下一个十年。

结语：这不是产品发布战，是计费逻辑战

2026 上半年 AI 代理领域最热闹的不是模型能力——三家的模型能力已经在 GPT-5.2、Claude Opus 4.6、Gemini 3.0 Pro 附近拉平；也不是产品数量——微软有 20+ Copilot、OpenAI 有 8+ SKU、Google 有 6+ Gemini 变体，产品数量的军备竞赛已经无意义。真正的战场是计费逻辑——谁能率先把「代理干活多少」翻译成「客户账单多少」，谁就赢得下一个十年的现金流曲线。

微软 Copilot 矩阵化 + Scout GA + $37B 年化 + credit 计费的组合，是第一份产业级答卷。OpenAI 的 ChatGPT 订阅 + Agent 融合、Google 的 Gemini Enterprise + Cloud 双轮，都是可行的另一种答卷。但至少在 2026-07-04 这一天，「代理商业化拐点」这四个字，第一次不再是 PPT 上的口号，而是财报里的数字、订阅页上的价格、以及企业 IT 预算表里第二条独立列。

代理战争才刚刚开始。第一场胜负手已经落地，下一场胜负手的关键——不在模型的参数，而在客户账单的每一美元里。
