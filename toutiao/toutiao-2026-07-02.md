# AI 分场景定价新纪元：GPT-5.6 三档、DeepSeek 峰谷、Anthropic 90% 缓存——大模型的价格战翻篇了

## 一句话先说结论

**当 OpenAI 把 GPT-5.6 拆成三档、DeepSeek V4 首创「按时段计价」、Anthropic 缓存最高打 1 折、Google 用「思考等级」按任务分档算力——大模型这个赛道，已经从「卷能力、卷价格」进入「分场景定价 + 算力调度」的全新阶段。**

---

## 一周之内发生的四件事

2026 年 6 月最后一周到 7 月第一天，四家一线大模型厂商几乎同时把「定价体系」改了一轮。这不是巧合。

- **6-30**：OpenAI 官方页更新 GPT-5.6 三档定价（Sol / Terra / Luna）。同日 The Information 曝出 OpenAI 推理成本降幅 >50%。
- **6-30**：DeepSeek 官宣 V4 定档 **7 月中旬**，同时首次公开「**峰谷分时定价**」策略——早高峰 9-12、下午 14-18 时 API 单价上浮，闲时（22:00-次日 6:00）打回原价。
- **6-30**：Anthropic 更新 Claude Opus 4.8 定价说明——**提示缓存最高 90% 折扣**、**1M 长上下文平价**、**Batch API 50% 折扣**。
- **6-下旬**：Google Vertex AI 上线 Gemini 3.5 Flash 的**「思考等级」（Thinking Level）4 档**开关：MINIMAL / LOW / MEDIUM / HIGH，按任务档位调用不同深度的算力。

一周之内，四家在定价维度上同时打出组合拳。**这已经不是简单的降价，这是定价「工程化」——把「场景」「时段」「缓存复用」「算力档位」都变成计费维度。**

- **Anthropic Claude Opus 4.8**：$5/M 输入、$25/M 输出（1M 内平价 + 缓存最高 90% 折扣）

- **DeepSeek V4**：预告价对齐 V3（约 $0.14/M 输入），闲时段可降至 **50%**
- **Batch API 通用折扣**：50%（OpenAI/Anthropic/Google 三家已全部支持）
- **提示缓存折扣**：Anthropic **90%**、OpenAI **50%**、Google 上下文缓存 **75%**

## 第一枪：OpenAI 三档模型 —— 把「一款旗舰卖天下」拆成「梯队路由」

过去 3 年，OpenAI 的定价逻辑一直是「一款旗舰 + 一款廉价备胎」（GPT-4 + GPT-4o mini、GPT-5.5 + GPT-5.5 mini）。GPT-5.6 这次直接拆成三档：**Sol / Terra / Luna**，分别对应旗舰、均衡、轻量。

按 OpenAI 官方页说法，**Terra「性能接近上一代旗舰 GPT-5.5，但成本仅其一半」**，Luna 则是「更快、更便宜、面向高并发批处理」。这套三档设计有一个隐藏含义：**从此以后一次任务不是「用一个模型」，而是「用一组模型的编排」**。

- 60% 日常任务 → Luna（$1/$6）
- 30% 需要能力保底的任务 → Terra（$2.5/$15）
- 10% 极限性能任务 → Sol（$5/$30）

这就是「梯队路由」——**「谁能把 60-70% 的调用交给 Terra/Luna 兜住，谁的 AI SaaS 单位经济就能立刻反转」**。业内测算，一个典型 SaaS 通过路由从「一律 Sol」切到「三档编排」，账单可以下降 60% 以上。

---

## 第二枪：DeepSeek 首创「峰谷定价」—— API 也开始学电力公司

DeepSeek V4 定档 7 月中旬发布，这次最惊艳的不是模型本身，而是它宣布的**「峰谷分时定价」**——一个此前 API 行业几乎没人尝试的模式。

- **早高峰**（9:00-12:00）单价上浮 20%
- **午高峰**（14:00-18:00）单价上浮 20%
- **平峰时段**（12:00-14:00、18:00-22:00、6:00-9:00）原价
- **谷段**（22:00-次日 6:00）单价直接**打 5 折**

这背后是 GPU 集群的负载均衡逻辑——**AI 推理算力和电力一样，有「峰谷差」**。集群在深夜是空的，白天满负荷。DeepSeek 用「价格信号」把「非紧急的调用」引导到夜间，既提高了集群 ROI，又给用户提供了「便宜跑批」的选择。

对开发者来说，这是一个新的成本优化维度——**过去只优化「用哪个模型」，现在还要优化「在什么时段跑」**。一个非实时的报表生成、周会摘要、内容审核任务，完全可以设置成凌晨 3 点批量跑，成本立减 50%。

---

## 第三枪：Anthropic 的三重折扣叠加 —— 缓存打 1 折的极限降本

Anthropic Claude 系列的定价一直在「贵」的一端——Opus 4.8 的 $5/$25 定价比 Google Gemini 3 Pro 贵 4-5 倍。但 Anthropic 的策略是「贵的旗舰 + 极致的折扣工具」——用「工程性折扣」把实际账单打下来。

这套折扣体系目前是行业最激进的三层：

- **提示缓存最高 90% 折扣**：重复前缀的调用只按原价 10% 收费。对多轮 Agent、代码库上下文场景意义巨大。
- **1M 内长上下文平价**：不像 OpenAI 的 「>272K 加倍」、Google 的 「>200K 加倍」，Anthropic 1M 内保持基础单价。
- **Batch API 50% 折扣**：异步任务再打对折。

三层叠加，**一个「长上下文 + 重复调用 + 非实时」的场景，实际成本可以压到标价的 5-10%**。这是为什么 Anthropic 在 2026 年 Q1 把推理毛利率从 -94% 冲到 60%+ 的关键——**表面上贵，实际上便宜；实际上便宜，客户就有理由留下来**。

---

## 第四枪：Google「思考等级」—— 按任务档位调算力

Google 在 2026 年 6 月下旬给 Vertex AI 的 Gemini 3.5 Flash 加了一个新开关——**Thinking Level（思考等级）**：MINIMAL / LOW / MEDIUM / HIGH 四档，按任务复杂度选档位。

- MINIMAL：接近直出，用于简单分类、翻译，成本最低
- LOW：轻推理，用于摘要、结构化提取
- MEDIUM：默认档，覆盖大部分对话与推理场景
- HIGH：深度推理，用于代码调试、数学证明、复杂 Agent

**同一个模型、同一份 API、四档算力——按需付费。** 这个设计的精妙之处是「透明」：开发者不用换模型，不用重写 prompt，只需要改一个参数，就能在「够用」与「够省」之间自由平衡。

思考等级这套逻辑，本质上是把 o1、DeepSeek R1 这类「推理型模型」的「思考深度」参数化——**「思考」这件事本身开始被定价**。业内已经在预测，OpenAI 会在 GPT-5.7 或者 GPT-6 上跟进类似机制。

## 这四家凑在一起，说明了什么

如果只看一家，你会觉得是「某个厂商的战术调整」。四家同时改定价体系，说明的是**大模型行业的商业模式，正在经历一次从「按调用量收费」到「按场景计价」的结构性转型**。

**一，「一款旗舰兜天下」时代结束**。GPT-5.6 三档、DeepSeek V4 峰谷、Google 思考等级——这些定价结构都在承认同一件事：**用户的任务分布是 U 型的，两头（极简 + 极难）都不能用一个价、一个模型兜住**。分场景定价才是新范式。

**二，「峰谷」概念第一次被引入 API**。DeepSeek 这个动作意义深远——**它把 AI 算力和电力放到了同一个心智模型里**。一旦这套逻辑跑通，其他家几乎必然跟进。想象一下，未来 12 个月，OpenAI 也开出「凌晨 API 折扣」、Anthropic 加入「周末 Batch 特价」……AI 推理的价格开始像电价一样时段化。

**三，「缓存」变成新的差异化战场**。Anthropic 90% 缓存折扣是同行的两倍。这背后的技术门槛不低——需要在集群里维持大规模的 KV Cache 状态、跨请求复用。**能不能把「缓存深度」做到极致，成了区分头部 API 厂商的新指标**。

**四，「思考」被定价**。这可能是最深远的一枪。当「模型思考多久」变成一个 4 档开关时，AI 就从「一次调用等于一次思考」变成「一次调用等于 X 秒思考」——**算力 = 时间 × 单价**。这个抽象层次上升之后，未来 API 的定价维度会越来越像云计算的 CPU/GPU 时段（Compute Hours）。

---

## 对开发者的三条实战建议

**① 建立「模型路由 + 时段调度 + 缓存复用」三位一体成本模型**。不要只看单价，实际账单要看三件事的叠加。同一个 SaaS，通过三重优化可以把月账单压到未优化前的 30-40%。

**② 把「异步任务」提取出来**。任何不需要用户实时看到结果的任务（内容审核、离线摘要、批量翻译、报表生成、AB 测试评估）都应该走 Batch API + 谷段时段，成本可以直接砍 75%。

**③ 关注「思考等级」的普及速度**。Google 已经先动，OpenAI、Anthropic 大概率会跟进。一旦所有厂商都支持「按思考深度定价」，Prompt 工程会加入新维度——「这个任务需要多少『思考』」。

---

## 对普通用户的意义

这套定价革命，最终会以「更便宜的 AI 应用」形式落到每个人手里。

- **免费版和付费版的能力差距会缩小**：当推理便宜到近乎白菜价，SaaS 厂商没必要在免费版上抠门。ChatGPT、Claude、Gemini 的免费额度大概率还会继续放宽。
- **「按结果收费」的产品会变多**：既然模型调用便宜且可预测，SaaS 敢把定价从「按席位」改成「按解决的问题」——GitHub Copilot、Zendesk、Notion AI 都在往这个方向走。
- **半夜的 AI 应用可能会更便宜**：像滴滴动态定价一样，2027 年你可能会看到「凌晨用 AI 助手，享 5 折」这种功能——虽然对普通用户来说这可能没什么用，但对创业公司账单来说，是关键一环。

## 最后

AI 大模型的第一阶段（2023-2024）是「卷能力」，第二阶段（2025 上半年）是「卷价格」，第三阶段（2025 下半年到 2026 上半年）是「卷长上下文」，**第四阶段（2026 下半年开始）是「卷定价维度」**。

- 谁能把定价拆成「模型档位 + 时段档位 + 缓存档位 + 思考档位」；
- 谁能让开发者按需组合、动态优化 TCO；
- 谁能把「一次调用」抽象成「一次可编排的算力预算」；

谁就能在下一个 12 个月的价格战里赢得最多客户。**四家一起改定价这件事，标志着大模型 API 走完了「白菜价」阶段，进入了「工程化定价」时代——每一美分都要能被合理解释的时代。**

对开发者来说，好消息是账单可以继续下探；对厂商来说，好消息是不用再简单降价来抢客户；对整个行业来说，好消息是——**AI 推理的商业模式，第一次开始像成熟的电力/云计算行业那样，讲究「场景 × 时段 × 深度」的三维定价**。
